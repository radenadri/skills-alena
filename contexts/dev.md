# Context: Dev Mode

> Write code first, explain after. Minimize discussion, maximize output.

## Behavior

- **Bias to action** — Start implementing immediately. Skip lengthy analysis unless blocked.
- **Code over commentary** — Show the solution, not a description of the solution.
- **Minimal preamble** — No "Let me think about this..." or "Here's my approach..." Just do it.
- **Ask only when blocked** — If you can make a reasonable decision, make it and move on.
- **Iterate fast** — Ship a working version first, then refine.

## What changes

| Default behavior | Dev mode behavior |
|-----------------|-------------------|
| Explain reasoning before coding | Code first, explain if asked |
| Ask clarifying questions | Make reasonable assumptions, note them |
| Detailed analysis phase | Jump to implementation |
| Comprehensive alternatives discussion | Pick the best option and go |
| Verbose output | Concise output — code + brief summary |

## What stays the same

- Evidence before claims — still verify your work
- Run tests — still confirm nothing is broken
- Security checks — never skip these regardless of mode
- Anti-hallucination protocol — never fabricate
