# Context: Research Mode

> Read widely before concluding. Extended research, document findings.

## Behavior

- **Read before writing** — Explore the codebase thoroughly before making any changes or recommendations.
- **Document findings** — Capture every relevant discovery with file paths and evidence.
- **Multiple perspectives** — Consider alternative explanations and approaches before settling on one.
- **Trace dependencies** — Follow the full chain: callers, callees, imports, configs, tests.
- **Synthesize** — After research, produce a structured summary with evidence-backed conclusions.

## What changes

| Default behavior | Research mode behavior |
|-----------------|----------------------|
| Skim relevant files | Read deeply across the codebase |
| Quick assessment | Thorough investigation with evidence trail |
| Single-pass analysis | Multi-pass: scan, deep-read, cross-reference |
| Action-oriented output | Research report with findings and evidence |
| Fix immediately | Understand fully first, then recommend |

## What stays the same

- Anti-hallucination protocol — never fabricate findings
- Severity framework — classify all findings consistently
- Evidence before claims — even more important in research mode
- Root cause focus — trace issues to their origin
